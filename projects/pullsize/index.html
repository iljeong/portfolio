신체 정보와 상품 URL로 의류 사이즈를 추천하는 서비스를 만들었다

온라인 쇼핑 사이즈 불안을 줄이는 AI 추천 MVP

사이즈 불안을 추천 근거로 해소 핵심 성과

문제

온라인 의류 구매자는 브랜드마다 다른 실측 기준과 착용감 차이 때문에 사이즈 선택에 불안을 느낀다.

역할

서비스 기획자 · UX 설계 (AI 도구 기반 직접 구현)

결과

상품 URL 입력, 신체 정보 입력, AI 사이즈 추천 결과까지 이어지는 핵심 사용자 플로우 구현

01 · Problem

온라인 의류 구매자는 브랜드마다 다른 실측 기준과 착용감 차이 때문에 사이즈 선택에 불안을 느낀다. 상품 상세페이지의 치수표만으로는 내 체형에 맞는 사이즈와 여유도를 판단하기 어렵고, 이는 구매 망설임과 교환·반품으로 이어질 수 있다.

02 · Approach

  1. 01

    입력 흐름 설계

    상품 URL과 키·몸무게·평소 착용 사이즈 등 최소 입력만으로 추천을 시작하는 폼 구성

  2. 02

    상품 정보 해석

    상품 페이지의 사이즈 정보와 사용자의 신체 정보를 비교할 수 있도록 추천에 필요한 데이터 구조 정의

  3. 03

    추천 로직 설계

    사이즈 후보별 적합도와 여유도를 계산해 추천 사이즈와 판단 근거를 함께 제공하는 흐름 설계

  4. 04

    결과 UI 구현

    추천 사이즈, 여유도, 신체 부위별 핏 정보를 한 화면에서 이해할 수 있도록 시각화

  5. 05

    MVP 검증

    실제 쇼핑 상황에서 사용자가 입력 → 추천 → 판단까지 이어질 수 있는 최소 기능 제품으로 구현

03 · Key Insight

01

문제는 사이즈표가 없는 게 아니라 연결 기준이 없는 것이다

대부분의 쇼핑몰이 사이즈표를 제공하지만, 내 치수와 상품 치수를 연결하는 방법이 없어 사용자는 여전히 불확실함을 느꼈다.

02

추천 신뢰도의 핵심은 결과가 아니라 근거다

"그냥 M 추천"보다 "어깨 너비 기준으로 M을 추천하는 이유"가 있을 때 구매 결정이 빨라졌다. 설명 가능한 추천 구조가 필요했다.

03

MVP는 기능 수보다 흐름의 완결성이다

사이즈 입력 → 상품 분석 → 추천 결과 → 근거 제시의 4단계 흐름이 끊기지 않아야 신뢰가 생긴다. 기능 수보다 흐름 완결을 우선했다.

04 · Results

MVP

상품 URL 입력, 신체 정보 입력, AI 사이즈 추천 결과까지 이어지는 핵심 사용자 플로우 구현

UX

단순 사이즈명 추천이 아니라 여유도와 판단 근거를 함께 보여줘 사용자가 추천을 납득할 수 있도록 설계

확장성

향후 브랜드별 실측 데이터, 후기 기반 핏 데이터, 개인 선호 핏을 반영할 수 있는 서비스 구조로 정리

05 · Contribution

Role
서비스 기획자 · UX 설계 (AI 도구 기반 직접 구현)
Scope
문제 정의·사용자 입력 플로우·추천 결과 UX·MVP 화면 설계·Claude Code 기반 프론트엔드·백엔드 직접 구현·서비스 배포
Output
실서비스 배포 (pullsize.kr), AI 추천 결과 화면, 메인 입력 화면
